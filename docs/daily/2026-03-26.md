### tsconfig.json の役割について
- `tsx` は `tsconfig.json` を参照しない（esbuild ベースで型チェックなしにトランスパイルする）
- `tsconfig.json` は VS Code の TypeScript Language Server が参照し、補完・エラー表示・Go to Definition 等を有効にするためのもの
- `"module": "NodeNext"` + `"moduleResolution": "NodeNext"` の意味を確認

### bindTools の役割
- `bindTools(tools)` は Claude API リクエストの `tools` パラメータにツール定義を付与するメソッド
- これにより LLM がテキストの代わりに `tool_calls` を返せるようになる

### `as AIMessage` 型アサーション
- `messages` 配列の要素は `BaseMessage` 型で `tool_calls` プロパティがない
- `as AIMessage` でコンパイラに型を伝え、`tool_calls` へのアクセスを可能にする
- 実行時の変換は行わない（プログラマーが実際の型を保証する必要がある）

### VS Code keybindings.json の問題調査・修正
- `cmd+enter` で Claude Code CLI の入力欄にフォーカスはされるが、選択テキストが送られない問題が発生
- 原因1: 「同じキー・同じ条件に複数バインドを定義すると最後のものだけが実行される」VS Code の仕様
- 原因2: `runCommands` 内でネストされた `args` の `${selectedText}` が展開されない VS Code のバグ
- 解決策: `runCommands` を使いつつ、`${selectedText}` の代わりにクリップボード経由（`clipboardCopyAction` → `terminal.focus` → `terminal.paste`）に変更
