## Phase 1 — MVP CLI

**期間**: Week 1（3/22 土 - 3/23 日）
**作業量**: 8h
**戦略**: 3/22（土）に動くところまで持っていく。3/23（日）はバッファとして翻訳品質の調整やバグ修正に充てる

**やること**:
1. プロジェクト初期設定（TypeScript, rss-parser）
2. 複数RSSソース取得 + クエリフィルタリング（HN / Zenn / dev.to）、`--limit` オプション対応
3. Claude API連携（**完全翻訳**・High/Mid/Low分類）、翻訳の並列化（`Promise.allSettled`）
4. Markdown出力 + CLI完成 + パフォーマンス計測

**完了基準**:
- `ai-news-agent "クエリ" --limit 10` で `./output/YYYY-MM-DD.md` が生成できる
- 完全翻訳の品質が満足できる
- 5記事で2分以内、10記事で3分以内

---

## Phase 2 — 内側エージェント化

**期間**: Week 2-3（3/29 土 - 4/6 日）
**作業量**: 16h（Week 2: Tool use・フォールバック、Week 3: embedding・スコアリング）

**やること**:
1. Tool use設計（fetchRSS / webSearch / fetchTechBlog / checkDuplicate / scoreArticle をLLMが呼び出す形に再設計）
2. **テックブログソースの追加**（Anthropic, OpenAI, Cohere 等）
3. **情報抽象度レイヤーに応じたソース選択ロジック実装**（マクロ/ミドル/ミクロ）
4. **フォールバック付き多段フェッチループの実装**（品質・量が不足したら追加ソースへ自動フォールバック）
5. Multi-Query実装（LLMが不足情報を自律的に追加検索するループ）
6. Web検索連携（Tavily API または Perplexity API）
7. embeddingによる重複除去（Voyage AI または OpenAI embedding + コサイン類似度）
8. LLM-as-judge による重要度スコアリング・品質評価（フォールバック判断も兼ねる）

**完了基準**: 重複記事が目視で明らかに減り、重要度順が体感で正しい。量・質が不足した日もフォールバックで補完されている。テックブログからの収集が機能している

---

## Phase 3 — MCPサーバー化・二重エージェント構造

**期間**: Week 4（4/12 土 - 4/13 日）
**作業量**: 8h

**やること**:
1. `@modelcontextprotocol/sdk` でMCPサーバーとして再実装
2. `fetch_ai_news` ツール（NewsRequestインターフェース）の公開
   - `query`, `abstractionLayer`, `targetSources`, `date`, `limit`, `userIntent`, `generateAudio`
3. **TUI対話フェーズの実装（Claude Code側）**
   - ユーザーとの対話で情報ニーズを明確化
   - 動的な追加質問生成
4. **abstractionLayerの判定ロジック実装**（対話結果からマクロ/ミドル/ミクロを判定）
5. Claude Code への登録・疎通確認
6. TUI対話 → 情報ニーズ明確化 → ニュース収集 → md生成が完結することを確認

**完了基準**: TUI上で対話 → 情報ニーズ明確化 → ニュース収集 → md生成が完結する。媒体・本数・トピックを柔軟に指示できる

## Phase 4 — 音声ダイジェスト（Phase 3 完了後）

**やること**:
- 音声用スクリプト生成ロジック（LLMで口語体要約を生成）

- 音声ファイル生成（MP3）
- md内への音声リンク埋め込み
- `generateAudio` フラグによる ON/OFF 制御

**完了基準**: 音声ダイジェストが生成され、朝の準備中や散歩中に聞ける品質になっている
