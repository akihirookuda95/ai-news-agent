## Phase 1 — MVP CLI

**期間**: 2026-03-11 〜 2026-03-25（約2週間）
**作業量**: 8h（2h × 4日）

**やること**:
1. プロジェクト初期設定（TypeScript, rss-parser）
2. 複数RSSソース取得 + クエリフィルタリング（HN / Zenn / dev.to）
3. Claude API連携（要約・翻訳・High/Mid/Low分類）
4. Markdown出力 + CLI完成

**完了基準**: `ai-news-agent "クエリ"` で `./output/YYYY-MM-DD.md` が生成できる

---

## Phase 2 — 内側エージェント化

**期間**: 2026-03-26 〜 2026-05-01（約5週間）
**作業量**: 20h（2h × 10日）
※ フォールバック戦略・抽象度レイヤー対応の追加により前回より拡張

**やること**:
1. Tool use設計（fetchRSS / webSearch / checkDuplicate / scoreArticle をLLMが呼び出す形に再設計）
2. **情報抽象度レイヤーに応じたソース選択ロジック実装**（マクロ/ミドル/ミクロ）
3. **フォールバック付き多段フェッチループの実装**（品質・量が不足したら追加ソースへ自動フォールバック）
4. Multi-Query実装（LLMが不足情報を自律的に追加検索するループ）
5. Web検索連携（Tavily API または Perplexity API）
6. embeddingによる重複除去（Voyage AI または OpenAI embedding + コサイン類似度）
7. LLM-as-judge による重要度スコアリング・品質評価（フォールバック判断も兼ねる）
8. エージェントループの統合テスト

**完了基準**: 重複記事が目視で明らかに減り、重要度順が体感で正しい。量・質が不足した日もフォールバックで補完されている

---

## Phase 3 — MCPサーバー化・二重エージェント構造

**期間**: 2026-05-02 〜 2026-05-20（約3週間）
**作業量**: 10h（2h × 5日）
※ TUI対話フェーズの実装が追加されたため前回より拡張

**やること**:
1. `@modelcontextprotocol/sdk` でMCPサーバーとして再実装
2. `fetch_ai_news` ツール（NewsRequestインターフェース）の公開
   - `query`, `abstractionLayer`, `targetSources`, `date`, `limit`, `userIntent`
3. **TUI対話フェーズの実装（Claude Code側）**
   - ユーザーとの対話で情報ニーズを明確化
   - 動的な追加質問生成
4. **abstractionLayerの判定ロジック実装**（対話結果からマクロ/ミドル/ミクロを判定）
5. `claude_desktop_config.json` への登録・Claude Codeとの疎通確認
6. TUI対話 → 情報ニーズ明確化 → ニュース収集 → md生成が完結することを確認

**完了基準**: TUI上で対話 → 情報ニーズ明確化 → ニュース収集 → md生成が完結する
